Cygnet Applications

This describes how to build applications in the Cygnet framework, in xml. A lot of common conventions are explained. Other dialog boxes, and pop ups get put into separate files, and will be dealt with later on.

The root element for each xml file is program, and the xml is validated via a public DTD. Each file starts with a meta information section, then the formats available to the program, then the layout of the application. The file contains details of who made the program, and the layout including menus, menu items, icons, large icons (with pictures and text on them) and links to any dialogs that need to pop up.

Commands are already contained within the Cygnet API, and are documented. These are for common things, such as data validation, opening and saving in various formats and printing. Plugins are written by the application creator or by Paradise Software, and implement extra functionality outside of the core API. Plugins are written in Python 3.3.

The meta information part has author information, and a description of the program. More than one author may be listed. The security section is for linking a md5 hash or key/certificate to the program. That is useful for checking for authencity and whether the program has been tampered with. A md5 hash is created by typing $ md5sum app_name.xml > hash_name.md5
at the Linux command line. The program is checked against the md5 hash by typing: $ md5sum -c hash_name.md5

File Formats

To enable a file format, you put the corresponding read/write format commands into the formats node of the xml. This calls the correct pieces of the API for reading and writing files. Some take mandatory options. See the table below.

write_open_document_word_files None odt, Libreoffice odt templates
read_open_document_word_files None odt, Libreoffice odt templates
write_open_document_spreadsheets None ods

write_open_document_presentations None odp presentations
read_open_document_presentations None odp files
write_scrolls_xml None My own file format, for use with books and long documents.

write_text_files file_ending (linux_or_mac/windows) txt files

write_ms_doc_files office-version (1997/2000/2003/2007/2010/2013) doc, docx that can be read in Libreoffice

Layout Order

The layout has various parts that need to be in a certain order. Each application has a menu bar under a title bar displaying the name of the program, a common bar for commonly used functions and a file port and status bar. The file port contains the opened file, while the status bar is for updates that refresh often.

The tool pane is optional, and mobile layouts won't have one. In the tool pane you can have tool boxes, which look like field sets, full of icons. Below the tool pane, there is a help/error message area called the help box.

Layout Tag Order

  1. menu_bar
  2. common_bar
  3. file_port
  4. status_bar
  5. tool_pane

Menu Bar Creation

Each menu has an acckey which underlines one key to be used with ALT, or displays the keyboard shortcut. A separate events program registers the shortcut, enabling its use.

Common Bar

The common bar stores large icons, for common tasks or useful tasks, say looking up a word in a dictionary.

These can also be controls with things like text fields within them.

Tool Pane

The tool pane holds 1 or more tool boxes, and the help box for information messages and small errors. The icons and controls are kept within a table, and look like this in the browser. The help box holds error, help and information messages, and tucks them out of the way. The more important errors will still pop up in a modal dialog.

File Port

The file port contains a lexigraphic checker, to check for spelling and grammar mistakes. A context menu is also available for right clicks anywhere in that space.

Status Bar

The settings menu on the left holds things like launchers for other applications, a menu for changing the theme and language changes for instance. The status bar holds small frequently updating programs, such as a word count. These provide text only output so as not to be so distracting.
