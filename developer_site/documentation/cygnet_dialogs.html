Dialog XML

For the cygnet applications you build, you will also need one or more dialog boxes. These are written in XML with a different DTD to validate against. The four different types of dialog box are:

  • quick_menu (with a show/hide option)
  • confirm (Simple yes/no question)
  • non-modal (can be moved or ignored, close button)
  • modal (User has to click ok or cancel before control is restored to the rest of the app)

The root element, 'dialog' has these options in the type attribute. There's also a name attribute for setting the dialog title. Each dialog box is a separate HTML file with CSS, done in a modular fashion. In each dialog, there can be any number of tabs, including zero, but there must be at least one field_box. Field boxes are used to group controls and buttons. They can take name, plugin, width and height options. Width and height are always in percent though.

In the field boxes, there must be 1 to 10 row elements, that are numbered. This is to line up things. See the example below, which is for picking special characters, such as a smiley face.
