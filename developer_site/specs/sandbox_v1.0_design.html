Cygnet Maker - Program

Definition:
Reads Cygnet files and outputs HTML, CSS, PHP, Javascript and Python files.

Responsibilities

  • To put the HTML and other code snippets in the right order.
  • To write standards compliant code that works across many browsers.
  • To encode SQL queries that work in any of the common databases, and that doesn't expose security risks.
  • To take in any options and check them for flaws (using a separate function).
  • To keep the appearence and structural CSS apart at all times.
  • To provide consistent methods of linking programs to options.
  • To provide specific methods of data validation for office and web applications, for example to prevent 'divide by zero' in maths.
  • To provide default JQuery animations for the modern web.

Constraints

There may be problems with the syntax initially.

Composition

  • data_validation (classes and methods)
  • options_checker (for each function, binary search)
  • reader (with syntax checker)
  • writer

Interactions

Reader outputs to the writer. Reader calls the options checker, the writer adds extra data validation calls to methods.

Inputs

A Cygnet text file with a simple syntax.

Processing

Use of token processing. Binary search. A bubble sort on the options checker.

Exports (Returns)

The web application in the folder it is ran in (the current working directory).
