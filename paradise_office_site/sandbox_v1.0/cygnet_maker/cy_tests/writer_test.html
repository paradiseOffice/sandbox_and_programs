Input: Cygnet XML files for the application (read only). Shelves, for commands, plugins and pictures (the URI's checked).

Output - A fully working application.

  • security_checker.php

    Checks the origins, and security key against the unprocessed Cygnet XML main_app file.

  • copy_skel_meta_info.php

    Copys the skeleton HTML and associated files, writes up the meta information in a similar way to this HTML file.

  • main_title_dialog_links.php

    The title of the app is set, and the opens_dialog XML attributes are converted into a list of links, and are grouped by place, then menu or icon. The folders for plugins are set up.

  • build_menus.php

    This reads the XML file for menus, and creates menus from it, using the links and shelve data (Python) provided. It puts the menu's in the menu bar last.

  • build_common_bar_icons.php

    This builds the large icons and controls, reading from the links and shelves, and then puts the icons into the common bar HTML.

  • build_tool_boxes.php

    This builds the icons and controls first, linking with any plugins, pictures from the Python data structures and then the icons are laid out, 4 in each row, numbered left to right. Each tool box is titled and added to the tool pane HTML. No processing needs to be done for the help box.

  • build_file_port.php

    This sets the file port type, and adds the context menu and sends options to the lexigraphic checker.

  • build_status_bar.php

    The settings menu comes with defaults already added. Any status bar plugins are inserted, and the updates set via PHP to refresh the status bar. Alternatively the updates could work via Javascript.

The test should be whether running this produces a fully working application. It's just combining web modules.
