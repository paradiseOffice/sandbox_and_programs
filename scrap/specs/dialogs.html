UI - Dialogs

Dialogs are different from quick menus in that they're a full, separate page to provide more room for controls and helpful explanations.

dialog page

A default dialog has the OK & Cancel buttons at the bottom of the page, separate from the fluid table based layout. OK submits, and closes the tab. Cancel merely closes the browser tab.

Field Box

A field box is one or more text inputs with labels to the left. These are in a group, with a title like 'address'. The default is three fields.

Font Preview

This displays either a sample of text displayed in that font, or a set of glyphs in that font with a list of different character sets e.g. cyrillic. Clicking on a glyph adds that text in unicode to the site. Selecting a font means the font on selected text is set to that font.

Pic Preview

If an uploaded picture URI has been passed in as an argument, this shows a small thumbnail of the picture, in a frame. Any comments in EXIF data could be used as the alt tag, which could be displayed in the tooltip.

Print View

This views the page as it would look if printed. More than one page can be viewed, and there are a lot of zooming tools. A print button to open the client OS print dialog should be on the top left. 'OK' should print a copy, cancel should just close that dialog page.

Tab

By default, the basic dialog comes with one tab. There can be up to 5 in a dialog page.

Text Preview

When the user uploads a text file this previews the first 5 lines and shows how it would look inserted after the cursor. Otherwise, this displays "no text file to insert".
