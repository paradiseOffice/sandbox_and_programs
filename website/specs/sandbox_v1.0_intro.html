Introduction

A description of the underlying API for the Paradise Office suite, called 'Cygnet'. How to use the API to build web applications. This document is intended for those with some programming knowledge. The Paradise Cygnet API is a set of functions written in common web programming languages to greatly simplify building complex web applications such as a word processor for instance. The programs are constructed using a web form and later via easy 'Scratch' style programming, which will convert into HTML, Javascript, and PHP with the relevant PDO SQL statements for interfacing with a variety of databases.

System Overview

The functions are grouped into areas such as data processing, UI elements and events.

Design Considerations

There aren't any issues except for maybe software patents held by big companies. Each piece of the system has to be separated out to ease development.

Assumptions and Dependencies

  • Either a Linux, Mac or Windows operating system.
  • A web browser with CSS3 support, HTML5 and Javascript
    • Firefox 16
    • Internet Explorer 9
    • Opera 11
    • Chrome (released in 2012 or after)
    • Safari (released in 2012 or after)
  • A screen resolution with a width of 420 pixels for mobiles, 500 pixels for tablets, and 700 pixels for desktops. Those are the minimums.
  • A computing device with at least a: 2GHZ CPU, 1024MB RAM, 1GB HDD, and a broadband internet connection.
  • A user account for Paradise Office.
  • People who've used software or the internet before.
  • Easy to alter pricing models or deploy the code on a companies internal servers.
  • Easy to add new functions to the API.
