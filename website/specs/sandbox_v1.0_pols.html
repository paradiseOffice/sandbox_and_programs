Policies and Tactics

Specific Products to Use

  • PHP 5.5.3 or higher
  • Postgresql 9.1
  • Apache 2.2 or Nginx.
  • OpenSSL (latest patched version)
  • Python 3.3
  • JQuery 1.10.2
  • JQuery UI latest stable version
  • A PHP/Javascript graphing library
  • Python lxml xml library and xsltproc
  • Firefox 22 to test everything in, with the Firebug debuggers.
  • Opera 11.60, Firefox 11, Internet Explorer 9, and common 'phones and tablets to test with.

Engineering Trade Offs

I will only convert the binary formats below for now.

  • Doc format (all these are MS 07, 2000, XP, 2003 formats)
  • XLS format
  • PPT format

Other binary formats will be converted from in later releases.

Coding Conventions Guidelines

Camel case variables, e.g. someVariable. CSS to have liberal comments breaking up sections, and one style per line. Blank lines to separate functions, and more comments in the PHP code. The XHTML style of coding to be used, instead of the laxer syntax of HTML4/5.

Where an OOP approach comes naturally to a program, it will be used. This means if the problem models real world objects and abstractions closely. Two spaces for the indentation in all code and markup. Curly braces to go on separate lines, except in an opening CSS rule block.

Protocols That Aren't HTTPS://

LDAP may be used to store calendar information and contacts. The POP, IMAP and SMTP servers may be connected to, to collect email. Plain HTTP will be used for the developer site, www.linux-paradise.co.uk. HTTPS will be used for www.paradise-office.co.uk.

Algorithms or Design Patterns

Planning for Requirements Tracebility

Each requirement for a program shall be read, and then manual test scripts shall be written e.g. to test a spellchecker throughly. As the script is being followed, any problems shall be written in a report with screengrabs or short videos also being used.

Testing the Software

This will be mostly manual (human) test plans, as each PHP module chunk shouldn't affect other parts of the system apart from underlying databases. The databases will also be checked where appropriate. Security testing will also be carried out, including trying to fake heavy traffic or DOS attacks.

Software Maintainence

New modules will be introduced for a particular problem, e.g. databases every month or so. Bug fixing will be the priority, with lots of testing being done on my own LAMP stack before it's uploaded to the live 'site.
